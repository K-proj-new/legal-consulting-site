# Content Structure README

## Назначение

Этот документ фиксирует текущую логику управления контентом сайта **Ayada Legal**.

Сейчас используется **Вариант 1**:
контент хранится в отдельных файлах внутри проекта, без backend и без настоящей CMS.

Это промежуточный и правильный этап перед переходом к полноценной админ-панели.

---

## Текущая структура

### Основной файл интерфейса
- `src/App.jsx`

В этом файле находится:
- layout страницы
- стили
- hero
- секции сайта
- рендеринг данных

---

## Данные, уже вынесенные из App.jsx

### Новости
- `src/data/news.js`

Экспорт:
- `news`

Назначение:
- хранение массива новостей для блока **Новости**

---

### Кейсы
- `src/data/cases.js`

Экспорт:
- `cases`

Назначение:
- хранение массива кейсов для блока **Кейсы**

---

## Что логично вынести следующим этапом

### Команда
Рекомендуемый файл:
- `src/data/team.js`

Экспорт:
- `team`

---

### Практики
Рекомендуемый файл:
- `src/data/practices.js`

Экспорт:
- `practices`

---

## Почему это важно

Такой подход даёт:

- более чистый `App.jsx`
- удобное редактирование контента
- понятную структуру проекта
- подготовку к следующему этапу: API / backend / admin panel

---

## Следующий этап развития

После завершения файловой структуры можно перейти к **Варианту 2**:

### Настоящая админ-панель
План:
- backend
- база данных
- авторизация
- CRUD для новостей / кейсов / команды / практик
- отдельный `/admin`

---

## Рекомендуемая последовательность

1. Вынести `team`
2. Вынести `practices`
3. Зафиксировать структуру в Git
4. Продолжить визуальную полировку
5. Потом перейти к форме заявок
6. Затем — к backend и админ-панели

---
